## 🐛 Проблема

При открытии `index.html` в браузере происходила бесконечная загрузка. Экран загрузки не исчезал, приложение не запускалось.

---

## 🔍 Причины (2 проблемы)

### 1. Синтаксическая ошибка (исправлена ранее)
```javascript
window.S = S;
};              // ❌ Лишняя закрывающая скобка
window.S=S;     // ❌ Дублирование
```

### 2. Firebase timeout (основная проблема)
Функция `loadCloud()` ожидала ответа от Firebase без таймаута. Если Firebase не отвечал или было медленное соединение, callback никогда не вызывался, и экран загрузки оставался навсегда.

```javascript
// БЫЛО:
loadCloud(()=>{
  refreshAll();
  // ... скрыть экран загрузки
});
// Если Firebase не отвечает - callback не вызывается = бесконечная загрузка
```

---

## ✅ Решение

Добавлен **таймаут 5 секунд** для загрузки из Firebase. Если Firebase не отвечает за 5 секунд, приложение продолжает загрузку с локальными данными.

```javascript
function loadCloud(cb){
  if(!fbOk){if(cb)cb();return;}

  // Защита от зависания
  let callbackCalled = false;
  const safeCallback = () => {
    if (!callbackCalled) {
      callbackCalled = true;
      if(cb)cb();
    }
  };

  // Таймаут 5 секунд
  setTimeout(() => {
    console.warn('Firebase load timeout - continuing with local data');
    safeCallback();
  }, 5000);

  // Попытка загрузки из Firebase
  firebase.database().ref('users/'+uid).once('value',snap=>{
    const d=snap.val();
    if(d){/* загрузка данных */}
    safeCallback();
  }).catch(()=>{safeCallback();});
}
```
